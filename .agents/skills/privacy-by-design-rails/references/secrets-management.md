# Secrets Management

> **Source:** https://guides.rubyonrails.org/security.html#custom-credentials

Never store encryption keys, API tokens, or secrets in source code or environment variables.

## Rails Credentials

```bash
# Generate AR Encryption keys
bin/rails db:encryption:init

# Edit encrypted credentials file
EDITOR="code --wait" bin/rails credentials:edit
```

The `credentials.yml.enc` file is safe to commit — it's encrypted with the master key in `config/master.key` (which is `.gitignore`d).

## Structure

```yaml
# credentials.yml.enc
active_record_encryption:
  primary_key: <generated>
  deterministic_key: <generated>
  key_derivation_salt: <generated>

secret_key_base: <auto-generated>
```

## Access in Code

```ruby
Rails.application.credentials.active_record_encryption.primary_key
Rails.application.credentials.secret_key_base
```

## Per-Environment Credentials

```bash
bin/rails credentials:edit --environment production
```

Creates `config/credentials/production.yml.enc` with its own `config/credentials/production.key`.

## Security Auditing Tools

**bundler-audit** — checks gems for known security vulnerabilities:

```bash
bundle-audit check --update
```

**Brakeman** — static analysis for Rails security issues (SQL injection, XSS, etc.):

```bash
brakeman
```

Both are already in the Gemfile under the `:development, :test` group. Run them in CI.

**pdscan** — scans databases for unencrypted PII columns (emails, phone numbers, IP addresses). Useful for auditing whether you've missed encrypting a sensitive field:

```bash
pdscan postgres://localhost/myapp_production
```

Install via `brew install ankane/brew/pdscan` or from [github.com/ankane/pdscan](https://github.com/ankane/pdscan).
